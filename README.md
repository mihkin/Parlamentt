# Parliament: PvE карточная политическая игра

Этот Unity-проект содержит прототип PvE-сцены карточной политической игры. Игрок и боты разыгрывают политические карты, набирают сторонников, копят ПО, атакуют соперников и могут запускать голосование за исключение.

## Быстрый запуск

1. Открой проект в Unity `2022.3.62f3` или совместимой версии 2022.3 LTS.
2. В верхнем меню Unity нажми `Parliament > Build PvE Scene`.
3. Открой `Assets/Scenes/PvEGameScene.unity`.
4. Нажми Play.

## Важные файлы

- `Assets/StreamingAssets/cards.json` - ЗДЕСЬ ДОБАВЛЯЮТСЯ НОВЫЕ КАРТЫ.
- `Assets/StreamingAssets/events.json` - здесь добавляются случайные события раунда.
- `Assets/Parliament/Scripts/Runtime/GameEnums.cs` - НЕ ЗАБУДЬ ОБНОВИТЬ ENUM, если добавляешь новый `type`, `targetType` или `effectType`.
- `Assets/Parliament/Scripts/Runtime/CardResolver.cs` - ЗДЕСЬ ОБРАБАТЫВАЮТСЯ ЭФФЕКТЫ.
- `Assets/Parliament/Scripts/Runtime/CardDefinition.cs` - структура карты, которую Unity читает из JSON.
- `Assets/Parliament/CardArtLibrary.asset` - библиотека иллюстраций карт, редактируется через Inspector.

ЭТО МОЖНО РЕДАКТИРОВАТЬ: `name`, `description`, `cost`, `value`, `duration`, `chance`, `rarity`, а также новые записи карт в `cards.json`.

## Как добавить новую карту

Открой `Assets/StreamingAssets/cards.json` и добавь новый объект в массив `cards`. `id` должен продолжать текущую нумерацию. Сейчас последняя карта имеет `id: 68`, значит следующая карта должна получить `id: 69`.

Пример:

```json
{
  "id": 69,
  "name": "Новая инициатива",
  "description": "Партия получает +3 сторонников.",
  "cost": 2,
  "type": "Influence",
  "targetType": "Self",
  "effectType": "AddInfluence",
  "value": 3,
  "duration": 0,
  "chance": 1.0,
  "rarity": "Common"
}
```

После изменения JSON перезапусти Play Mode. Файл должен оставаться в UTF-8, чтобы русские названия и описания нормально отображались в Unity.

## Как добавить картинку на карту

На prefab карты добавлено место под иллюстрацию. Картинки привязываются не в JSON, а через Inspector:

1. Импортируй картинку в Unity, например в `Assets/Parliament/CardArt`.
2. Выбери файл картинки в Project.
3. В Inspector поставь `Texture Type: Sprite (2D and UI)`.
4. Нажми `Apply`.
5. Открой `Assets/Parliament/CardArtLibrary.asset`.
6. В списке `Card Art` добавь новый элемент.
7. В `Card Id` укажи `id` карты из `cards.json`.
8. В `Sprite` перетащи импортированную картинку.

Если у карты нет своей картинки, на ней останется нейтральная область с подписью `Иллюстрация`. Можно также назначить `Default Sprite` в `CardArtLibrary.asset`, и тогда он будет использоваться для всех карт без отдельной картинки.

Если пересобираешь сцену через `Parliament > Build PvE Scene`, сборщик сохранит `CardArtLibrary.asset` и снова привяжет его к `UIManager`.

## Структура cards.json

`cards.json` хранит объект с массивом:

```json
{
  "cards": [
    { "id": 1, "name": "Пропаганда" }
  ]
}
```

Каждая карта обязана иметь все поля:

- `id` - уникальный номер карты.
- `name` - название карты на русском.
- `description` - понятное описание эффекта на русском.
- `cost` - стоимость розыгрыша карты в очках действия (`ОД`). Если у карты `cost: 3`, на ее розыгрыш нужно потратить 3 ОД текущего хода.
- `type` - категория карты.
- `targetType` - кто может стать целью.
- `effectType` - какой эффект применит `CardResolver`.
- `value` - сила эффекта: сколько сторонников/ПО добавить, снять, украсть или наносить каждый раунд.
- `duration` - длительность временного эффекта в раундах.
- `chance` - шанс срабатывания от `0.0` до `1.0`.
- `rarity` - редкость карты.

## Типы карт

- `Influence` - карты роста сторонников.
- `Politics` - ПО, добор карт и политическая подготовка.
- `Attack` - вред противникам.
- `Defense` - защита от голосований и контроля.
- `Neutral` - работа с нейтральными сторонниками.
- `Vote` - карты голосования и исключения.
- `Event` - рискованные или массовые события.
- `Boost` - усиление следующего действия.

## targetType

- `Self` - цель это тот, кто играет карту.
- `SingleEnemy` - один выбранный противник.
- `AnyParticipant` - любой активный участник.
- `AllEnemies` - все активные противники игрока карты.
- `Everyone` - все активные участники, включая автора карты.
- `RandomEnemy` - случайный активный противник.
- `None` - цель не нужна.

## effectType

- `AddInfluence` - мгновенно добавляет `value` сторонников.
- `RemoveInfluence` - мгновенно снимает `value` сторонников.
- `StealInfluence` - мгновенно забирает `value` сторонников у цели и передает автору карты.
- `AddPoliticalPoints` - мгновенно добавляет `value` ПО.
- `RemovePoliticalPoints` - мгновенно снимает `value` ПО.
- `DrawCard` - мгновенно добирает `value` карт, но не выше лимита руки.
- `StartEliminationVote` - мгновенно запускает голосование за исключение цели.
- `ProtectFromVote` - дает защиту от следующего голосования.
- `SkipTurn` - цель пропускает следующий ход.
- `MultiplyNextInfluenceGain` - умножает следующий прирост сторонников автора карты.
- `RandomInfluenceChange` - мгновенно случайно меняет сторонников в диапазоне от `-value` до `+value`.
- `PoisonInfluence` - наносит вред по времени: цель теряет `value` сторонников в конце каждого раунда.
- `PoisonPoliticalPoints` - наносит вред по времени: цель теряет `value` ПО в конце каждого раунда.

## Мгновенные и длительные эффекты

Мгновенные эффекты срабатывают сразу при розыгрыше карты: `AddInfluence`, `RemoveInfluence`, `StealInfluence`, `AddPoliticalPoints`, `RemovePoliticalPoints`, `DrawCard`, `StartEliminationVote`, `ProtectFromVote`, `SkipTurn`, `MultiplyNextInfluenceGain`, `RandomInfluenceChange`.

Урон по времени наносят только `PoisonInfluence` и `PoisonPoliticalPoints`. Они используют `duration`: если `duration` равен `3`, эффект сработает в конце трех раундов.

## Как работает chance

`chance` - вероятность срабатывания карты:

- `1.0` - карта всегда работает.
- `0.75` - карта срабатывает в 75% случаев.
- `0.5` - рискованная карта: половина попыток провалится.

Если карта не сработала, она все равно считается разыгранной и удаляется из руки.

## Как работает rarity

- `Common` - простые слабые карты.
- `Uncommon` - обычные полезные карты.
- `Rare` - сильные карты, контроль, высокий урон, кража сторонников.
- `Legendary` - очень мощные карты стоимостью 6+ или с большим риском/наградой.

Редкость не меняет механику сама по себе. Это метка для баланса, будущей выдачи карт и визуального оформления.

## Стартовая раздача

При первой раздаче каждый участник всегда получает одну случайную карту с `cost: 1`, чтобы в первом раунде у него точно было действие на 1 ОД. Остальные стартовые карты добираются случайно из всей колоды.

## Как балансировать стоимость

Стоимость карты теперь означает не цену покупки за ПО, а цену действия за розыгрыш. В начале каждого хода участник получает запас ОД, равный номеру текущего раунда:

- раунд 1 - 1 ОД;
- раунд 2 - 2 ОД;
- раунд 3 - 3 ОД;
- раунд 4 - 4 ОД;
- дальше по той же логике.

Когда карта разыграна, ее `cost` вычитается из ОД текущего хода. Если ОД не хватает, карту нельзя сыграть. Купленные или добранные в этом раунде карты остаются заблокированы до следующего раунда.

Используй такие ориентиры:

- `cost 1` - слабый эффект: +1-2 сторонников, +1 ПО, добор 1 карты, небольшой вред.
- `cost 2-3` - обычный эффект: +3-6 сторонников, 2-4 урона, кража 2-5 сторонников, слабый яд.
- `cost 4-5` - сильный эффект: +8-12 сторонников, массовый вред, пропуск хода, сильный яд.
- `cost 6+` - редкие мощные карты: крупная кража, сильный массовый яд, голосование, большой риск/награда.

Слишком сильные карты делай `Rare` или `Legendary`. Если карта может резко решить партию, добавь риск через `chance` ниже `1.0`.

## Контроль игры через Inspector

Основные настройки вынесены в компонент `GameManager` на объекте `GameManager` в сцене:

- `Player Turn Seconds` - сколько секунд длится ход игрока.
- `Bot Delay Min/Max` - задержка перед ходом бота.
- `Starting Political Points` - стартовые ПО каждого участника.
- `Starting Influence` - стартовый процент сторонников каждого участника.
- `Starting Hand Size` - сколько карт получает участник в начале партии.
- `Max Hand Size` - максимальный размер руки.
- `Round Political Point Gain` - сколько ПО получают активные участники за новый раунд.
- `Draw Card Cost` - цена ручного добора карты игроком.
- `Vote Cost` - цена голосования за исключение.
- `Round Event Chance` - шанс случайного события после полного раунда.
- `Bot Turn Seconds` - сколько секунд показывается таймер хода бота. Бот не ждет весь таймер: после розыгрыша доступных карт он завершает ход.
- `Bot Target Hand Size` - до какого размера руки бот пытается добирать карты.
- `Bots Spend Political Points To Draw` - тратят ли боты ПО на добор.
- `Visible Log Entries` - сколько строк истории показывать в UI.

В компоненте `UIManager` можно назначить `Card Art Library`, если хочешь использовать другую библиотеку иллюстраций.

## Сильные и слабые карты

Слабая карта обычно дает небольшой гарантированный эффект и стоит 1-2. Например: +2 сторонников, +1 ПО, снять 2 сторонников.

Средняя карта стоит 2-3 и заметно меняет темп: добор 2 карт, кража 4-5 сторонников, яд на 2-3 раунда.

Сильная карта стоит 4-5 и должна быть `Rare`: массовая атака, большой прирост сторонников, пропуск хода, сильный дебафф.

Легендарная карта стоит 6+ и должна быть редкой по смыслу: мощное голосование, большой перехват сторонников, сильное усиление следующего прироста.

## Как создать новый эффект

1. Добавь имя эффекта в `EffectType` в `GameEnums.cs`. НЕ ЗАБУДЬ ОБНОВИТЬ ENUM.
2. Добавь `case` в `CardResolver.ResolveCard` или во внутренний метод обработки эффекта. ЗДЕСЬ ОБРАБАТЫВАЮТСЯ ЭФФЕКТЫ.
3. Если эффект должен работать у событий, обнови `CardResolver.ResolveEvent`.
4. Добавь описание нового эффекта в этот README.
5. Только после этого используй новое имя в `cards.json`.

Если забыть шаг 1 или 2, Unity прочитает JSON, но карта не будет работать так, как ожидается.

## Голосование

Голосование исключает цель, если голосов `за` больше, чем `против`. Защита `ProtectFromVote` отменяет следующее голосование против защищенного участника.

Карты с `StartEliminationVote` запускают голосование сразу. Такие карты должны быть дорогими, `Rare` или `Legendary`, потому что они могут резко изменить партию.

## Кириллица и UTF-8

Все русские строки в JSON, README и C# должны сохраняться в UTF-8. Если в Unity видны квадратики вместо букв, проверь TMP Font Asset и диапазон кириллицы `0400-04FF`.
