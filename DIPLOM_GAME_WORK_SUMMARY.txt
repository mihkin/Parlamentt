PARLAMB — ПОЛНОЕ ОПИСАНИЕ ВЫПОЛНЕННОЙ РАБОТЫ ПО ПРОЕКТУ

Ниже приведено общее описание того, что было реализовано в игре ParlamB на текущем этапе разработки. Текст можно использовать как основу для диплома, отчета по проекту, пояснительной записки, описания функционала и перечня выполненных работ.

1. ОБЩАЯ ИДЕЯ ПРОЕКТА

ParlamB — это карточная политическая игра, разработанная на Unity. Игрок управляет собственной политической фигурой, собирает колоду, использует карты влияния, проводит голосования, получает монеты, открывает новые карты, улучшает свою коллекцию и может играть как против ботов, так и в сетевом режиме.

Основная концепция проекта:
- карточная пошаговая стратегия;
- тематика политической борьбы и влияния;
- развитие профиля игрока;
- коллекционирование карт;
- магазин наборов;
- одиночная игра против ИИ;
- онлайн-лобби и подготовка сетевой игры;
- использование собственной базы данных и backend API.

2. ТЕХНОЛОГИЧЕСКАЯ ОСНОВА ПРОЕКТА

В проекте использованы:
- Unity 2022 для клиентской части;
- C# для игровой логики и интерфейса;
- TextMeshPro для отображения текста;
- собственная UI-система на основе спрайтов, плашек и динамической генерации интерфейса;
- JSON-файлы для хранения базы карт и событий;
- ASP.NET Core backend API;
- SQL Server для хранения профилей, авторизации, лобби и сетевого состояния;
- JWT-аутентификация;
- локальное и серверное хранение данных профиля.

3. СТРУКТУРА ИГРЫ И СЦЕНЫ

В проекте подготовлены основные сцены:
- MainMenuScene — главное меню игры;
- PvEGameScene — одиночная игра против ботов;
- LanGameScene — сетевая/онлайн-матч сцена;
- SampleScene — тестовая сцена.

Также реализованы bootstrap-компоненты для автоматической настройки сцены и UI:
- MainMenuSceneBootstrap;
- LanGameSceneBootstrap;
- PersistentRuntimeServices.

4. СИСТЕМА КАРТ И ИГРОВЫХ ДАННЫХ

В игре реализована полноценная data-driven система карт:
- база карт хранится в cards.json;
- база событий хранится в events.json;
- загрузка данных выполняется через JsonDatabase;
- для работы с картами используются CardDefinition, CardDatabase, CardArtLibrary;
- поддерживается назначение артов карт через библиотеку спрайтов;
- реализована fallback-система, если карточная база в asset-форме не заполнена.

Каждая карта содержит:
- идентификатор;
- название;
- описание;
- стоимость;
- редкость;
- тип;
- игровой эффект;
- числовое значение эффекта;
- шанс и длительность действия, если это необходимо.

5. ИГРОВАЯ ЛОГИКА

В проекте реализована базовая пошаговая карточная логика:
- создание и хранение состояния партии через GameState;
- хранение состояния участника через ParticipantState;
- управление матчем через GameManager;
- логирование действий через ActionLog;
- пошаговая система ходов;
- контроль раундов;
- система очков действия;
- система влияния;
- обработка голосований;
- проверка победы и поражения;
- стартовая раздача карт;
- добор карт;
- использование карт игроком;
- обработка эффектов карт через CardResolver;
- применение событий во время игры;
- блокировки, временные эффекты и базовые состояния участников.

В игровой логике реализованы:
- покупка карты за очки действия;
- завершение хода;
- начало и проведение голосования;
- выбор цели для карты;
- расчет влияния;
- перерасчет нейтрального влияния;
- отслеживание текущего участника;
- результат матча;
- журнал игровых событий.

6. ИГРА ПРОТИВ БОТОВ

Реализован режим PvE:
- создана сцена PvEGameScene;
- добавлен BotAI;
